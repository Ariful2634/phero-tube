Discuss the scope of var, let, and const.

Ans: var, let, and const are three different ways to declare variables in JavaScript. They have distinct scopes and behaviors,
which impact how they can be used and accessed within your code.

1: var
  • var defined variables are function-scoped, which means that only the function in which they are declared can access them.
    They become globally scoped if defined outside of any function.
  • The global or function scope is where var declarations are placed. This means that even though a variable can be accessed
    and used before it is declared, its value is unknown.

2: let
  • Variables declared with let are block-scoped, meaning they are accessible only within the block (curly braces) in which they are defined.
  • You cannot access let variables prior to the line on which they are declared because they are not hoisted to the top of their block.

3: const
  • Variables declared with const are also block-scoped, just like let.
  • When a const variable is declared, it must be given a value that cannot be modified afterwards. A constant reference to the value is created.

Tell us the use cases of null and undefined

In JavaScript, null and undefined are two distinct values that indicate the absence of a meaningful value. They are often used in different
scenarios based on their characteristics.


1: undefined
  • A variable that has been declared but not given a value is represented by the word undefined. For variables that haven't been initialised,
    it serves as the default value.
  • It can also indicate the absence of a return value from a function or the absence of a specified property in an object.

2: null
  • An explicit value called null stands for the purposeful absence of any object value. It's frequently employed when you wish to make
    it clear that a variable or attribute purposefully lacks any value.
  • null is often used when you want to reset an object reference, effectively saying that the variable points to no object.

What do you mean by REST API?

Application programming interfaces, or APIs, are a collection of guidelines that specify how devices and software can connect to and communicate with one another. An API that adheres to the representational state transfer, or REST, design principles is known
as a REST API.
